# 系譜データ生成ブリーフ（サブエージェント共通）

トヨタ車系譜ビューアの樹形図を「日本国内で発売された全トヨタ車＋レクサス」へ全史化する。
あなたは **1カテゴリ分の担当**として、そのカテゴリに属するネームプレート（車名）の系譜データを
リサーチして JSON フラグメントを1ファイル出力する。日本語で作業すること。

## 精度水準

- **水準 B（リアルなデモ）**：日本語 Wikipedia・toyota.jp・GAZOO 等を一旦信頼し、正確性の厳密考証は課さない。
- 発売年（世代）は西暦の**フルモデルチェンジ年**。マイナーチェンジは含めない。多少の年ズレは許容。
- WebSearch / WebFetch を使ってよい（使えない場合は自分の知識で水準 B として補う）。

## 収録範囲

- **日本国内で正規販売された車種のみ**。海外専売車は除外
  （例の除外: Etios, Innova, Avanza, Fortuner, Venza, Sequoia, Tundra, 北米/中国専売の世代など）。
  国内名があるものは国内名で収録（例: クルーガー=国内名、ハイランダーは除外）。

## 出力スキーマ（1件 = 1ネームプレート）

```jsonc
{
  "label": "カローラ", // 車名（一意の識別子。派生参照キー）
  "sub": "1966–現行", // 補足（"開始年–現行" / "開始年–終了年" / "派生 / 親名"）
  "category": "<担当カテゴリID>", // 下記のうち自分の担当ID固定
  "status": "active", // 国内で新車販売中=active / 終了=discontinued
  "currentYear": 2019, // active のとき最新世代年（generations に含まれる値）
  "derivativeOf": "ランドクルーザー", // 任意。明確に他車から派生独立した場合のみ
  "branchYear": 1990, // derivativeOf とセット（枝分かれ年）
  "generations": [1966, 1972, 1979, 1987, 2000, 2012, 2019], // 各世代の発売年・昇順・重複なし
}
```

- **overviews は付けない**（後続パスで充填）。
- `derivativeOf` は無理に付けない。親が別カテゴリでも構わない（オーケストレータが解決する）。
- `currentYear` は status=active のときのみ。必ず `generations` に含まれる年にする。
- `generations` は**必ず昇順・重複なし・1件以上**。

## カテゴリID一覧（担当は割当で1つ指定される）

- `sedan` セダン：4ドアセダン/ハードトップ（乗用）。例: カローラ, クラウン, カムリ, マークII, マークX, コロナ, カリーナ, チェイサー, クレスタ, ビスタ, プレミオ, アリオン, プログレ, ブレビス, センチュリー, SAI, ミライ ほか
- `coupe-sports` クーペ・スポーツ：クーペ/スポーツ/オープン。例: 2000GT, セリカ, スープラ, ソアラ, MR2, MR-S, 86, GR86, GRスープラ, セリカXX, カローラレビン, スプリンタートレノ, パセオ, セラ, カレン, GRヤリス ほか
- `suv` SUV・クロカン：SUV/クロカン/クロスオーバー。例: ランドクルーザー, ランドクルーザープラド, ランドクルーザー70, RAV4, ハリアー, C-HR, ライズ, ヤリスクロス, カローラクロス, FJクルーザー, ハイラックスサーフ, ラッシュ, クルーガー, bZ4X ほか
- `minivan-wagon` ミニバン・ワゴン：ミニバン/トールワゴン/ステーションワゴン（乗用）。例: アルファード, ヴェルファイア, ノア, ヴォクシー, エスクァイア, エスティマ, シエンタ, ウィッシュ, イプサム, ガイア, ラウム, ポルテ, スペイド, ルーミー, タンク, カルディナ, マークIIブリット, ラウム ほか
- `compact` コンパクト・ハッチ：コンパクト/ハッチバック（登録車・乗用）。例: ヴィッツ, ヤリス, パッソ, アクア, プリウス, プリウスPHV, スターレット, ist, ラクティス, カローラII, ターセル, コルサ, iQ, ピクシス以外の小型 ほか
- `kei` 軽自動車：軽（多くは近年のダイハツ OEM）。例: ピクシス エポック, ピクシス メガ, ピクシス スペース, ピクシス ジョイ, ピクシス バン, ピクシス トラック, スパーキー(該当すれば) ほか
- `commercial` 商用・トラック・バス：バン/トラック/バス/商用。例: ハイエース, レジアスエース, ハイラックス(ピックアップ), ダイナ, トヨエース, タウンエース, ライトエース, プロボックス, サクシード, コースター, ジャパンタクシー, コンフォート, ハイメディック ほか
- `lexus` レクサス：レクサス全車。例: LS, GS, IS, ES, HS, CT, RC, RC F, LC, SC, UX, NX, RX, GX, LX, RZ, LM, LFA ほか

## 粒度ルール（重複を防ぐ）

- **別ネームプレート**＝独立車名。サブネーム/ボディ違い（例: カローラ ツーリング/アクシオ/フィールダー、ノア/ヴォクシー）は原則**同一ネームプレートの世代**として扱い独立させない。
  - ただし歴史的に独立車名として広く定着したもの（レビン と トレノ、マークII と チェイサー と クレスタ、ノア と ヴォクシー 等）は**独立ネームプレート**として別々に収録してよい。
- 自分の担当カテゴリに**明確に属する車種だけ**を収録する。境界が曖昧な車種は無理に取り込まない（オーケストレータが最終調整）。
- 同一車種を2カテゴリで重複して出さない。迷ったら sub に理由を書き、最も代表的なカテゴリにのみ置く。

## 出力

- ファイル: `data/raw/genealogy-parts/<担当カテゴリID>.json`
- 中身: `Nameplate` オブジェクトの**JSON 配列**（トップレベルが `[ ... ]`）。
- 30件前後を上限に、そのカテゴリの主要〜代表車種を網羅する（マニアックすぎる派生まで追わなくてよいが、有名どころは漏らさない）。
- 書き終えたら、収録した label 一覧と件数を1行で報告して終了する。
